# Phase 1.2: Tokenomic Mechanisms Inventory

## Overview

This document catalogs all existing and proposed tokenomic mechanisms within Regen Network, assessing their automation potential.

## Active Mechanisms (On-Chain)

### M001: Staking Mechanism
| Attribute | Value |
|-----------|-------|
| **Module** | x/staking |
| **Purpose** | Secure network, enable governance |
| **Token Flow** | Holder → Validator → Rewards |
| **Current State** | Active, 75 validators |
| **Automation Potential** | High (monitoring, alerts) |

### M002: Governance Voting
| Attribute | Value |
|-----------|-------|
| **Module** | x/gov |
| **Purpose** | Decentralized decision-making |
| **Token Flow** | Staked weight → Vote power |
| **Current State** | Active, 61+ proposals |
| **Parameters** | 7-day voting, quorum-based |
| **Automation Potential** | Medium (analysis, not voting) |

### M003: Transaction Fees
| Attribute | Value |
|-----------|-------|
| **Module** | x/auth, x/bank |
| **Purpose** | Spam prevention, validator rewards |
| **Token Flow** | Sender → Validators |
| **Current State** | Active |
| **Automation Potential** | Low (parameter monitoring only) |

### M004: Marketplace Escrow
| Attribute | Value |
|-----------|-------|
| **Module** | x/ecocredit/marketplace |
| **Purpose** | Trustless credit trading |
| **Token Flow** | Buyer → Escrow → Seller |
| **Current State** | Active, 23+ orders |
| **Currencies** | USDC.grv, USDC.axl, EEUR, REGEN |
| **Automation Potential** | High (monitoring, curation) |

### M005: Community Pool Treasury
| Attribute | Value |
|-----------|-------|
| **Module** | x/distribution |
| **Purpose** | Fund ecosystem development |
| **Token Flow** | Block rewards → Pool → Recipients |
| **Current State** | ~3.2M REGEN |
| **Automation Potential** | Low (human decisions) |

### M006: Credit Class Creator Allowlist
| Attribute | Value |
|-----------|-------|
| **Module** | x/ecocredit |
| **Purpose** | Quality control for registries |
| **Token Flow** | Governance proposal deposit |
| **Current State** | Active, permissioned |
| **Automation Potential** | High (screening, recommendation) |

### M007: Currency Allow List
| Attribute | Value |
|-----------|-------|
| **Module** | x/ecocredit/marketplace |
| **Purpose** | Control accepted payment tokens |
| **Token Flow** | Governance proposal deposit |
| **Current State** | 4 currencies enabled |
| **Automation Potential** | High (criteria evaluation) |

## Proposed Mechanisms (New)

### M008: Data Attestation Bonding
| Attribute | Proposed Value |
|-----------|----------------|
| **Purpose** | Incentivize accurate ecological data |
| **Token Flow** | Attester → Bond Pool → Release/Slash |
| **Participants** | Attesters, Challengers, Arbiters |
| **Bond Range** | 500-5,000 REGEN by attestation type |
| **Automation Potential** | High (monitoring, challenge detection) |

### M009: Service Provision Escrow
| Attribute | Proposed Value |
|-----------|----------------|
| **Purpose** | Trustless service engagement |
| **Token Flow** | Client → Escrow → Milestones → Provider |
| **Participants** | Clients, Providers, Arbiters |
| **Service Types** | Verification, methodology dev, MRV |
| **Automation Potential** | Medium (milestone tracking) |

### M010: Reputation/Legitimacy Signaling
| Attribute | Proposed Value |
|-----------|----------------|
| **Purpose** | Stake-weighted endorsement system |
| **Token Flow** | No transfer (weight from stake) |
| **Participants** | Signalers, Subjects, Aggregator |
| **Categories** | Quality, Legitimacy, Competence, Rigor |
| **Automation Potential** | High (aggregation, monitoring) |

## Dependencies for New Mechanisms

### M008: Data Attestation Bonding
- Requires: IRI resolution infrastructure
- Depends on: DAO DAO for arbiter DAOs
- Integration: x/data module for attestation storage

### M009: Service Provision Escrow
- Requires: CosmWasm deployment
- Depends on: DAO DAO for dispute resolution
- Integration: x/ecocredit for credit-related services

### M010: Reputation Signaling
- Requires: Off-chain aggregation layer
- Depends on: x/staking for stake weights
- Integration: On-chain anchoring of scores
